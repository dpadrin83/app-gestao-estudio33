Profissionais E33 + banco de prompts

Três formas de organizar no Hub o que o estúdio já fazia nos prints (posicionamento, identidade, aplicações…) e o que o Briefing Studio faz para soluções digitais (PRD + prompt de construção). Cada sub-etapa tem dias e prompt. Você cria áreas macro (ex.: Redação copy) e lista o trabalho dentro delas.

Como você descreveu (modelo mental)

1. Área macro — nome do bloco de trabalho · ex.: Redação copy
  ↳ 2. Sub-etapas — passos internos, cada um com dias · ex.: Pesquisa (1d), Leitura do briefing (1d), Criação dos textos (3d)
  ↳ Opcional: 3. Entregável ao cliente — o que sai no portal · ex.: Copy site aprovado

Isso é diferente da fase do Hub (planejamento → entrega): a macro é como você organiza o trabalho; a fase é onde o projeto está no cronograma.
Base de dados sugerida:
project_macro_areas (id, project_id, nome, professional_id, sort_order) · macro_work_items (tipo: sub_etapa | entregavel, nome, estimated_days, professional_id) · professionals + lista consolidada (catálogo fixo E33) — campo Área profissional no formulário de entregável · soma dos dias → cronograma.

Versão 1 — Área macro + sub-etapas (seu exemplo)

Recomendada para o Hub

Três ações: + Nova área macro · + Sub-etapa · + Entregável (nome + área profissional do catálogo + dias).

1
+ Nova área macroNome do bloco (ex.: Redação copy). Profissional padrão opcional.
2
+ Sub-etapaNa macro selecionada: Pesquisa, Briefing… Nome + dias.
3
+ EntregávelNome (ex.: Assinatura de e-mail) + Área profissional (lista dos 11) + dias.

Exemplo prompt vinculado a “Análise SWOT”:
Analise [CLIENTE] no segmento [SEGMENTO]. Matriz SWOT com 5 itens por quadrante, linguagem executiva, sem jargão vazio. Formato: tabela + 3 implicações estratégicas.

Vantagens

  • 100% familiar — igual proposta comercial antiga
  • Cliente reconhece o pacote

Atenção

  • Lista longa; digital (DEV) fica à parte
  • Muitos prompts para cadastrar (1 por linha)

Versão 3 — Tabela mestre: etapa × entregável × dias × prompt
